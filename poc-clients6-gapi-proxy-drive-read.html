clients6.google.com GAPI Root Proxy — Direct postMessage Attack (v2)

Root cause: clients6.google.com/static/proxy.html reads #parent, #rpctoken, and #origin from its own URL hash — all attacker-controlled. All three makeHttpRequests checks pass because we set all three hash values.

v1 bug: gapi.client.request() made a DIRECT CORS request from this page (referrer = this origin), hitting the API key HTTP referrer restriction.
v2 fix: We send makeHttpRequests DIRECTLY to the proxy via raw postMessage. The proxy makes the XHR from clients6.google.com (Google's own origin) — the API key referrer restriction does NOT apply.

To test: Open in a browser logged into Google. Drive files appear within ~4 seconds. Works from ANY origin (github.io, localhost, sites.google.com, etc.)
