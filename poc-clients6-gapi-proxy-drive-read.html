clients6.google.com GAPI Root Proxy — Parent Injection

Bug: clients6.google.com/static/proxy.html reads #parent and #rpctoken from its own URL hash — attacker-controlled. GAPI makeHttpRequests validates these values (which we set) instead of verifying the actual calling origin against an allowlist.
Result: any page can trigger authenticated same-origin XHR to clients6.google.com/* using the victim's Google cookies → Drive file read, and potentially other APIs.

To test: Open this page in a browser where you are logged into Google. If the attack works, your Drive file list will appear below within ~5 seconds.
